Mytra Internship · Transition Runbook

Useful in one hour. Powerful after that.

The first objective is a verified company Git identity, a working editor, and one employer-authenticated AI client. Custom orchestration comes only after work and personal state are proven separate.

First-Day Critical Path

0–10 min
Clone dotfiles, run ./setup --profile work, install command-line prerequisites.
10–20 min
Set work Git identity and authenticate the employer GitHub account.
20–35 min
Start Bash, Neovim, Python tools, and validate a disposable repository.
35–50 min
Authenticate the employer-supported AI client and confirm the active account.
50–60 min
Clone a company repository, read its instructions, run its tests, and stop if identity checks fail.

Full Work Profile

  • Ghostty + AeroSpace
  • Homebrew Bash + Starship + zoxide
  • Neovim/LazyVim with a strong Python baseline
  • Codex, OpenCode, Claude Code, and Cursor
  • Profile-approved MCP integrations

Minimal Recovery Profile

  • Apple Terminal + Zsh
  • Neovim, Git, ripgrep, fd, Python
  • No window manager requirement
  • No external integration requirement
  • Designed to remain usable under policy or setup failures

Before Company Code

  • Git email is the work identity
  • GitHub CLI shows the employer account
  • AI client shows the employer account
  • Work configuration root is active
  • No personal MCP or browser profile is connected
  • Repository-local agent instructions were reviewed

Hard Stops

  • An AI client silently falls back to a personal login.
  • A company repository resolves a personal Git identity.
  • A remote integration's data access is unclear.
  • A setup log prints a token or credential.
  • Company content appears inside this dotfiles repository.

After the Baseline

Add Ghostty and AeroSpace polish, authenticate the remaining AI clients, then connect GitHub, documentation, browser automation, Slack, Notion, and internal systems one at a time. Each connection needs an owner, documented read/write scope, credential location, and revocation path.
